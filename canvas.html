    UX Laws Analysis: Canvas LMS Redesign

    Canvas is the course management system used at Columbia University. When I first started using Canvas, I found its interface frustrating: the platform feels cluttered.

    By using Hick’s Law, we simplified decision-making by reducing the number of choices presented at any given moment. Progressive Disclosure allowed us to reveal information only when necessary, preventing users from being bombarded with irrelevant details upfront. Additionally, optimizing the Signal-to-Noise Ratio helped eliminate unnecessary elements, ensuring that essential content remained prominent and easily accessible.

    1. Course List Redesign (Information Hierarchy & User Recognition)

    Problem: Semester banners and course numbers took up too much space, and important information (class times, assignments) was not easily accessible.

    UX Principles Applied:

    • Recognition Over Recall – Uses class times instead of course numbers for easier recognition.
    • Von Restorff Effect – Highlights urgent assignments for immediate attention.
    • Pareto Principle (80/20 Rule) – Focuses on the most-used features first.

    Solution: Moved semester banners to a dropdown, displayed class times instead of course numbers, and bolded “Next Assignment” deadlines.

    2. Individual Course Page Redesign (Navigation & Content Organization)

    Problem: Confusing structure with unclear distinctions between "Home," "Course Info," and "Syllabus." Assignments were buried, requiring extra clicks.

    UX Principles Applied:

    • Least Surprise Principle – Matches navigation to user expectations.
    • Goal-Gradient Effect – Prioritizes assignments upfront to boost task completion.
    • Gestalt’s Principle of Proximity – Groups related course details together for clarity.

    Solution: Made "Assignments" the default landing page, created a dedicated syllabus page, and organized content logically.

    3. Minimizing Complexity

    Guiding UX Laws:

    • Tesler’s Law of Complexity – Managed inherent complexity by simplifying the interface.
    • Miller’s Law (7±2 Rule) – Limited the amount of information displayed at once.
    • Dieter Rams’ Principle – "Good design is as little design as possible."

    Outcome: Transformed Canvas into a user-centered platform with intuitive navigation, reduced friction, and clearer task flows.

    Final Takeaway:

    By applying these UX principles, the redesign shifts Canvas from a high-friction system to an efficient, student-friendly learning tool. The result is a minimalist, goal-oriented interface that streamlines the learning experience.
